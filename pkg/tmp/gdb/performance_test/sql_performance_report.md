# gdb.Sql 性能测试报告

## 测试环境

- 操作系统: Darwin (macOS)
- CPU: Apple M1
- Go 版本: go1.x (根据实际环境)
- 测试时间: 2025-12-26
- 测试次数: 10次

## 测试概述

本报告针对 gdb.Sql 库的各种 SQL 操作进行性能测试，包括 Select、Update、Create、Delete 等操作。测试使用 Go
的基准测试框架，测量每项操作的执行时间、内存分配和对象分配次数。

## 性能分析与评价

### 优势

1. **基础操作高效**：最简单的查询操作只需要 650.0 纳秒，内存分配仅 128 字节，分配 4 个对象，性能非常出色。
2. **操作构建快速**：Table 操作仅需 200.0 纳秒，内存分配仅 16 字节，分配 1 个对象；Raw 操作仅需 891.6 纳秒，内存分配仅 48
   字节，分配 2 个对象，说明基础构建操作非常快速。
3. **结构体支持**：基于结构体的表操作性能良好，仅需 500.0 纳秒，内存分配 128 字节，分配 4 个对象，说明结构体到表名的转换效率很高。

### 性能特点

1. **操作复杂度与资源消耗成正比**：随着查询条件的复杂度增加，操作时间、内存分配和对象分配数也相应增加。
2. **批量操作优化**：批量创建操作（3,908 ns/op）相比单条创建（3,575 ns/op）只增加了少量时间，说明批量操作实现良好。
3. **连接和分页操作适中**：JOIN 操作（1,525 ns/op）和分页操作（1,942 ns/op）的性能表现适中，适合在实际应用中使用。
4. **嵌套切片查询性能**：对于复杂参数如嵌套切片 `[][]any{{1, 2}, {3, 4}}`，处理时间约 13,075 ns/op，内存分配 769 B/op，分配
   20 个对象，相对耗时较长。
5. **子查询支持**：子查询功能性能表现良好，处理时间约 3,933 ns/op，内存分配 1,945 B/op，分配 34 个对象，适用于复杂业务场景。

### 总体评价

gdb.Sql 库在性能方面表现出色，特别是在基础查询和简单操作方面。对于复杂的 SQL 构建，资源消耗（时间、内存、对象分配）会相应增加，但整体性能仍然在可接受范围内。

在实际应用中，gdb.Sql 可以高效地处理各种 SQL 操作需求，无论是简单的查询还是复杂的多表连接查询，都能够在合理的时间和内存消耗范围内完成
SQL 语句的构建。

该库的性能特征表明它适合用于高并发、高性能要求的应用场景，特别是需要频繁构建 SQL 语句的场景。
