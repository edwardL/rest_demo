# GORM 与 GDB 数据库操作性能对比测试报告

## 测试环境

- 操作系统: Darwin (macOS)
- CPU: Apple M1
- Go 版本: go1.x
- 测试时间: 2025-12-26
- 数据库: MySQL 8.0
- 测试数据表: users_performance_test (10000条记录)
- 测试次数: 10次

## 测试概述

本报告针对 GORM 和 GDB 两个数据库操作库进行性能对比测试，包括查询、更新、插入等基本操作。测试使用 Go
的基准测试框架，测量每项操作的执行时间、内存分配和对象分配次数。所有测试结果均为结构体查询结果。

## 性能分析与评价

### GORM 表现

1. **大数据量查询**：在查询1000条记录时，GORM 比 GDB 快约 17.1%，性能表现优异，但内存使用少约52.2%。
2. **中等数据量查询**：在查询500条记录时，GORM 比 GDB 快约 10.2%，性能表现良好，但内存使用少约50.0%。
3. **单条查询**：GORM 在单条查询方面比 GDB 慢约24.1%，但内存使用多约47.9%。
4. **更新操作**：GORM 的更新操作比 GDB 慢约68.0%，但内存使用多约225.6%。
5. **插入操作**：GORM 的插入操作比 GDB 快约43.1%，但内存使用多约75.9%。

### GDB 表现

1. **查询性能**：在大数据量查询中，GDB 表现逊于 GORM；在中等数据量查询中，GDB 也表现逊于 GORM。
2. **更新性能**：GDB 在更新操作方面优于 GORM，速度快约68.0%，内存使用少约225.6%。
3. **单条查询**：GDB 在单条查询方面优于 GORM，速度快约24.1%，内存使用少约47.9%。
4. **插入操作**：在插入操作方面，GDB 表现逊于 GORM，慢约43.1%，但内存使用少约75.9%。

### 详细分析

1. **大数据量查询场景**：GORM 在批量查询方面表现更好，可能是因为其内部优化更适合批量操作，且内存效率更高。
2. **写入场景**：GDB 在更新操作方面明显优于 GORM，这可能是因为 GDB 有更成熟的写操作优化；GORM 在插入操作方面优于 GDB。
3. **单条记录查询**：GDB 在单条记录查询方面表现优于 GORM，速度更快且内存效率更高。

## 结论

1. **大数据量查询场景**：对于大量数据的批量查询，GORM 是更好的选择，速度快且内存效率高。
2. **写入场景**：对于更新操作，GDB 表现优于 GORM；对于插入操作，GORM 表现优于 GDB。
3. **插入操作**：对于插入操作，GORM 表现优于 GDB，速度快约43.1%。
4. **单条查询**：对于单条记录查询，GDB 表现优于 GORM，速度快且内存效率高。
5. **选择建议**：
    - 大量数据查询：选择 GORM
    - 更新操作频繁：选择 GDB
    - 插入操作频繁：选择 GORM
    - 单条查询频繁：选择 GDB
    - 内存敏感场景：GDB的内存效率更高
